# Day19 步骤规划目录说明

- 目录：`day19回归/步骤规划/`
- 创建日期：`2026-03-04`
- 目的：沉淀 Day19 每个 Step 的“具体做法文档”，统一执行口径，避免信息分散在聊天记录中。

---

## 1. 使用规则（冻结）

1. Day19 每个 Step 都在本目录新增一个独立文档。
2. 文档命名统一为：`Day19_{Phase}_{Step}_{主题}_v1.0.md`。
3. 每份文档至少包含以下固定章节：
   - 目标与要解决的问题
   - 范围与非目标
   - 统一字段口径（如果该 Step 涉及观测/指标）
   - 详细执行步骤（可复现）
   - DoD 验收映射
   - 对项目收益与风险说明
4. 文档中的命令、接口、SQL 必须可在个人项目环境复跑，不写不可执行伪步骤。
5. 设计更新时，升级版本号（`v1.1`、`v2.0`），不直接覆盖历史结论。
