## 3. Day02 API 规则

1. `auth.ts` 只保留登录 / 注册 / 激活职责；资料编辑和账号安全不回流到 `auth.ts`。
2. 资料变更成功后，若需要刷新本地 session，必须通过统一 helper 回写，不能在多个页面各自维护 localStorage。
3. 地址列表默认按分页接口消费，新增 / 编辑共用表单时也必须保留提交路径区分。
4. 头像上传按“配置获取 -> 实际上传 -> 资料回写”三段式记录；任一段未跑通都不得写成完成。
5. API 模块命名、返回值适配和错误提示都必须为后续回填留出证据路径。
