# UserFrontDay03 范围冻结（Scope Freeze）

- 日期：`2026-03-18`
- 文档版本：`v1.0`
- 当前状态：`计划中（v1.0 已建档，执行未开始）`
- 当日主题：`市场浏览、详情、评论与举报`

---

## 1. 起点结论

- 结论：`UserFrontDay03` 应作为第一批计划中的市场域正式入口。
- 判定依据：
  1. 后端已有 `MarketProductController`、`FavoriteController`、`ReviewController`，说明市场域并非空白；
  2. 当前用户端前端还没有列表、详情、收藏、评论与举报页面，必须先有独立 Day 才能稳定推进；
  3. 市场域与账户域、卖家域、订单域边界都不同，不宜混在 Day02 或 Day04 中处理。

---

## 2. Day03 要做什么

1. 冻结市场商品列表、详情、筛选、分页与评论列表的页面 / API / 回填边界。
2. 冻结商品举报的用户侧入口、失败态记录与留证方式。
3. 冻结收藏 / 取消收藏 / 收藏状态 / 收藏列表的页面与 API 模块边界。
4. 为“我的评价入口”预留真实台账，并在需要时记录与订单前置关系。
5. 规划 Day03 的路由、联调准备与验收标准。

---

## 3. Day03 不做什么

1. 不吸收用户商品管理、卖家工作台、订单、钱包、积分等其他业务域。
2. 不把控制器存在或 DTO 可推断，直接写成前端已实现。
3. 不把评论列表和评价提交混写成同一条已完成链路；若前置条件不足，必须保留阻塞说明。
4. 不把 Day03 进度回写到 `demo-admin-ui`。

---

## 4. Day03 范围拆解

| 范围分类 | 归属内容 | Day03 处理方式 |
|---|---|---|
| 市场浏览与详情 | `GET /user/market/products`、`GET /user/market/products/{id}` | 作为 Day03 主链，冻结列表、详情、筛选与分页边界。 |
| 评论与举报 | 评论列表、举报提交、我的评价入口 | 作为 Day03 主链，但评价提交若受订单前置影响，必须写明阻塞。 |
| 收藏夹 | 收藏状态、收藏 / 取消收藏、收藏列表 | 作为 Day03 主链，与市场列表 / 详情协同规划。 |
| 沿用基线 | Day01 的 auth/session/router 基线 | 只做复用说明，不重写 Day01 结论。 |
| 非 Day03 范围 | 卖家工作台、用户商品、订单、钱包、积分 | 留给 Day04 以后处理。 |

---

## 5. Day03 未来验收口径

> 以下是 Day03 真正进入执行后才需要满足的收口口径；当前 v1.0 仅完成计划建档，不代表已经满足。

1. 市场列表、详情、评论 / 举报、收藏三组子流都有明确 owner、接口口径和回填入口；
2. 列表 -> 详情 -> 收藏 / 举报 / 评论的导航关系清晰可执行；
3. 若评价提交受外部前置限制，文档必须诚实记录阻塞而不是虚报完成；
4. 构建、运行、联调状态都能在 `05_进度回填` 中找到证据；
5. 覆盖矩阵与 Day03 文档结论保持一致。

## 6. Day03 证据口径

- 当前已具备的证据：`MarketProductController.java`、`FavoriteController.java`、`ReviewController.java`、Day03 六文档骨架。
- 当前尚未具备的证据：Day03 相关前端页面、API 模块、构建结果、运行态或联调结果。
- 因此 Day03 当前只能写到 `文档已记录`，不能写成 `代码已确认 + 构建已通过` 或 `联调已通过`。
