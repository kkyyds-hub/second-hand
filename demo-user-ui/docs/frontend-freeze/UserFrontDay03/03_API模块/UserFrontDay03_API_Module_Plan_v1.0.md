## 3. Day03 API 规则

1. `market.ts` 只处理市场商品浏览域；`userProducts.ts` 留给 Day04，不要提前混合。
2. 收藏状态、收藏动作与收藏列表由 `favorite.ts` 统一维护，避免多个页面各自拼 URL。
3. 举报和评价属于显式提交动作，API 层必须保留失败信息，不能只返回成功 toast。
4. 列表、详情与评论数据的字段适配在 API 层集中处理，不把转换逻辑散到多个页面。
5. 任何仍未验证的评论提交前置条件都必须写回文档，不通过页面兜底掩盖真实依赖。
