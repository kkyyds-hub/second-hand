## 1. 对齐目标

围绕 `用户商品管理与卖家工作台扩展`，冻结 Day04 需要承接的卖家摘要、用户商品 CRUD 与状态流转接口入口，并明确当前还没有任何对应前端实现落地。

## 3. Day04 接口对齐原则

1. `seller.ts` 继续承接卖家摘要；用户商品 CRUD 与状态流转单独进入 `userProducts.ts`，不把卖家域混进 `market.ts`。
2. 卖家工作台和用户商品页都必须复用 Day01 的鉴权与 401 规则，不另起认证逻辑。
3. 商品状态按钮文案与后端语义必须一致，尤其 `on-shelf` 兼容路径不能误写成“直接上架到 on_sale”。
4. 列表、详情、表单和状态流转都需要保留失败态与回填证据，不能只写成功流程。
5. 任何字段级不确定项都必须写成 `待执行时确认`，不能先按主观猜测固化成事实。
