## 3. Day04 API 规则

1. 卖家摘要继续走 `seller.ts`；商品 CRUD 与状态流转统一进入 `userProducts.ts`。
2. 列表、详情、表单与状态按钮的字段适配都在 API 层集中处理，不分散到多个页面。
3. `off-shelf`、`resubmit`、`on-shelf`、`withdraw` 都必须保留语义说明和失败态提示。
4. 任何需要依赖卖家身份的页面都必须复用 Day01 / Day02 已确认的 session 信息，不各自维护第二套身份判断。
5. 未经真实执行，不把 `userProducts.ts`、路由扩展或页面实现写成已完成。
