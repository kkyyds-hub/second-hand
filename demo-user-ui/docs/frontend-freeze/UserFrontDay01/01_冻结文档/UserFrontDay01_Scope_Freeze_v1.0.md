## 1. 起点结论

- 结论：`UserFrontDay01` 应作为用户端当前执行日正式启用。
- 判定依据：
  1. 仓库内已经存在真实主工程 `demo-user-ui`，且具备 `npm install / npm run dev / npm run build` 的标准前端入口；
  2. 当前工程已经具备 `request`、`router`、`UserLayout`、登录 / 注册 / 激活 / 退出 / 首页 / 账户中心等最小用户端壳；

## 2. Day01 要做什么

1. 冻结 `demo-user-ui` 的独立主工程边界、目录入口和构建入口。
2. 冻结 `src/utils/request.ts`、`src/router/index.ts`、`src/layouts/UserLayout.vue` 的职责边界。
3. 冻结登录 / 手机注册 / 邮箱注册 / 邮箱激活 / 退出登录 / 首页卖家摘要 / 账户中心基础展示的最小契约口径。
4. 建立 `demo-user-ui/docs/frontend-freeze/` 主入口、覆盖矩阵与 `UserFrontDay01` 六个文档入口。
5. 为后续地址、收藏、市场浏览、用户商品、订单、钱包、积分等业务域明确“暂未进入 Day01”的边界。

---

## 3. Day01 不做什么

1. 不在 Day01 直接进入地址、收藏、订单、钱包、积分等新业务页面实现。
2. 不把构建成功、页面存在或代码可见，直接写成“联调已通过”。
3. 不把用户端未实现能力写到 `demo-admin-ui` 的 `FrontDay` 文档里。
4. 不在页面 UI 中直接暴露 `UserFrontDay01`、freeze、mock/real 之类研发过程文案。
