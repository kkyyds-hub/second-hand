## 4. 结果回填区

1. 场景 A/B 关键返回：
   - `code=1, data=用户封禁成功`
   - `code=1, data=用户已处于封禁状态`
   - `code=1, data=用户解封成功`
   - `code=1, data=用户已处于正常状态`
2. 场景 C 关键返回：
   - 并发支付：`code=1, data=支付成功`
   - 并发取消：`code=0, msg=订单已支付，当前不允许取消（后续走退款/售后）`
3. 场景 D 关键返回：
   - 错密前 5 次：`code=0, msg=用户名或密码错误`
   - 第 6 次及后续正密：`code=0, msg=账号已被暂时冻结，请稍后再试或联系管理员`
4. SQL 核验结果：
   - 并发封禁后：`users.id=2 status=banned`
   - 并发解封后：`users.id=2 status=active`
   - 并发支付/取消后：`orders.id=900044 status=paid`
   - 风控冻结后：`users.id=2 status=frozen`
   - `user_bans` 中 `source='AUTO_RISK'` 记录增量：`+1`
5. 收尾清理：
   - 执行 `PUT /admin/user/2/unban` 后，SQL 最终状态恢复 `active`。
