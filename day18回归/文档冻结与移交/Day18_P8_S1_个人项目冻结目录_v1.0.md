## 3. 关键结论（可展示版）

1. 一致性路线采用“本地事务 + Outbox + 幂等消费 + 任务补偿”，未引入 Seata/Saga。
2. 并发控制采用 CAS 分流为主，悲观锁仅保留在必要资金链路。
3. 安全防护优先落在“敏感信息不泄露、输入安全、JWT 架构下 CSRF 边界明确”。
4. 性能治理聚焦“索引回归 + 热点入口保护 + 缓存策略规范”。
5. Day18 回归与安全扫描均有实测证据，满足学习项目可展示要求。
