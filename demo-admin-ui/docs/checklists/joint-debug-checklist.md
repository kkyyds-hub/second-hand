# 前后端联调验收清单

## 登录链路

- 打开管理端时，未登录用户会被正确跳转到登录页。
- 使用管理员手机号或邮箱可以成功登录。
- 登录成功后会跳转到首页。
- 本地 token 失效后，再访问受保护页面会被拉回登录页。

## 首页

- 首页可以正常打开，不白屏、不报路由错误。
- 首页在接口失败时仍能稳定展示兜底内容。
- 首页加载状态和普通展示状态切换正常。
- 页面布局在桌面端和窄屏下都没有明显错位。

## 用户与商家管理

- 用户列表能成功请求真实数据。
- 关键字搜索可以触发后端查询。
- 角色筛选可以触发后端查询。
- 状态筛选可以触发后端查询。
- 分页切换后，列表数据和总数联动正常。
- 每页条数切换后，列表会重新拉取。

## 人工建档

- 打开人工建档弹窗后可以正常输入昵称、手机号、角色。
- 提交后会调用真实后端创建接口。
- 创建成功后列表会自动刷新。
- 非法输入或接口失败时会有明确反馈。

## 封禁与解封

- 点击“限制账号”能正常打开封禁理由弹窗。
- 输入理由后提交，会把 reason 正确传给后端。
- 封禁成功后列表状态会更新。
- 已封禁用户可以执行解封。
- 解封成功后列表状态会恢复。

## 导航与占位

- 已完成页面都可以从左侧菜单正常进入。
- 未完成页面不会出现“能点但什么都没有”的体验。
- 页面切换时没有明显布局抖动或残留状态异常。

## 纠纷与违规

- 左侧菜单点击“纠纷与违规”后可以正常进入页面。
- 页面在真实接口返回时能正常展示统计卡和工单列表。
- 筛选条件变更后会重新拉取后端总览接口。
- 点击“查看详情”可以正常打开详情弹窗。
- 页面在没有数据时能正常显示空状态。

## 回归建议

- 每次改动登录、路由、请求拦截器后，都至少回归一次登录链路。
- 每次改动列表页交互后，都至少回归一次分页、筛选、弹窗提交流程。
- 每次补一个新页面后，都更新对应文档，记录当前是否已经接入真实接口。
