## 1. 模块目标

把当天相关的页面能力落到明确的 API 文件、字段映射、错误处理和 mock/real 策略上。

---

## 2. 重点文件

| 文件 | 角色 | 当日要求 |
|---|---|---|
| `demo-admin-ui/src/api/auth.ts` | 登录请求入口 | 保持登录参数与后端登录接口一致。 |
| `demo-admin-ui/src/utils/request.ts` | token 存储、请求头注入、401 清理 | 不得改成 `Authorization` 模式。 |
| `demo-admin-ui/src/router/index.ts` | 登录页与受保护页面路由守卫 | 未登录回跳逻辑在这里冻结。 |
| `demo-admin-ui/src/layouts/MainLayout.vue` | 左侧菜单和退出登录入口 | 菜单可达性以此为准。 |

---

## 3. 当日 API 规则

1. 登录态相关能力统一通过 `src/utils/request.ts` 管理，不分散到页面里。
2. 退出登录当前以清理本地 token 为准，不在页面里造第二套逻辑。
3. 导航入口变更时，必须同步更新根 README 的当前基线说明。

---

## 4. 预期输出

1. 接口口径与页面行为对应关系明确。
2. 字段映射、兜底、错误处理位置清晰。
3. 后续若改代码，可直接定位到当天关联文件。
