# 前端待办清单

## 页面建设

- 完成商品审核页基础骨架，包括筛选区、列表区、空状态。`已完成`
- 完成纠纷与违规页基础骨架，包括工单列表、状态标签、处理入口。`已完成`
- 完成系统设置页基础骨架，至少先具备可访问页面与分组布局。`已完成`
- 为暂未落地的菜单项补“建设中”占位态，避免点击后无反馈。

## 数据接入

- 核对首页指标卡对应的后端接口，逐步替换当前展示型 mock 数据。
- 核对首页待审核商品区块的数据来源，确认是否已有管理端聚合接口。
- 核对首页风控预警和纠纷区域的接口可用性，决定继续接真实接口还是保留临时 mock。
- 完成纠纷与违规页聚合接口接入，确认真实数据可由单接口返回。`已完成代码接入，待运行态联调确认`
- 检查所有管理端请求的异常提示是否足够明确，避免接口失败时页面静默。

## 交互统一

- 统一弹窗标题、正文间距、底部按钮布局。
- 统一表格空状态、加载态、错误态展示方式。
- 统一表单提交中的按钮禁用和 loading 表现。
- 统一操作成功后的刷新策略，例如刷新当前页还是回到第一页。

## 登录与鉴权

- 复查登录后 token 持久化逻辑是否符合当前管理端使用习惯。
- 验证 token 失效、清空、本地残留等场景是否都能正确回到登录页。
- 评估是否需要补“退出登录”入口，方便联调时切换账号。

## 联调与验收

- 按页面梳理一轮真实接口覆盖率，标记哪些区域已接真数据，哪些仍是兜底数据。
- 在关键页面补充一轮手工回归，确保导航、筛选、分页、弹窗类流程都稳定。
- 补一轮纠纷与违规页的运行态联调，确认本地服务启动后能正常返回真实数据。
- 每完成一个页面，及时更新文档，避免后续重复排查。
